# Examples

Real CodeTracer output, generated by running the tool **on its own source** (so every example is
reproducible from this repo). Each is committed exactly as produced (`gemma4:latest`, CPU). A
curated, "everything on" set — one full **explain**, one full **trace**, plus the **DI** showcase.

`explain` shows **each method's real source** under its heading (indented by call-depth, so the
nesting is visible) followed by the explanation — code *and* prose, the way a newcomer needs it.
Every result ends with an auto-generated **`## Call-flow`** diagram — a high-level map of what the
analysis found, drawn as **ASCII** (readable in any viewer) **and** **Mermaid** (renders as
graphics on GitHub / VS Code). It's deterministic (no extra model call).

| file | what it shows | command |
|---|---|---|
| [`explain-full-example.md`](explain-full-example.md) | **The full `explain` showcase** — deep + wide (12 methods, 3 levels) from `Agent.RunAsync` down. Each method's **real source** (indented by call-depth) + a step-by-step explanation, then an end-to-end synthesis, an **"In plain words"** recap, and the **Call-flow** diagram. Everything a newcomer needs, in one read. | `explain --method "Agent.RunAsync" --depth 3 --max-methods 12` |
| [`trace-full-example.md`](trace-full-example.md) | **The full `trace` showcase** — the path with the **code between every hop**, a **"why" note per hop** (`--annotate`), a final **Summary** + **"In plain words"** recap (`--summary`), and the **Call-flow** diagram (boxes + Mermaid). | `trace -f RoslynIndex.cs -e Agent.cs --with-bodies --annotate --summary` |
| [`trace-di-multiple-impls.md`](trace-di-multiple-impls.md) | **DI / interfaces, multiple implementations** — a call through an `INotifier` interface with 3 impls (+ a decorator); `--all-paths` lists all 5 distinct paths, and the **Call-flow** draws the fan-out as a branching tree. From [`../samples/di-playground`](../samples/di-playground). | `trace --from "NotificationService.Notify" --to "Audit.Record" --all-paths --no-llm` |

Each example notes its real run-time and token usage. See [`../MANUAL.md`](../MANUAL.md) for the
full reference.
