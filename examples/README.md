# Examples

Real CodeTracer output, generated by running the tool **on its own source** (so every example
is reproducible from this repo). Each is committed exactly as produced (`gemma4:latest`, CPU).

`explain` shows **each method's real source** under its heading (indented by call-depth, so the
nesting is visible) followed by the explanation — code *and* prose, the way a newcomer needs it.
Every result ends with an auto-generated **`## Call-flow`** diagram — a high-level map of what the
analysis found, drawn as **ASCII** (readable in any viewer) **and** **Mermaid** (renders as
graphics on GitHub / VS Code). It's deterministic (no extra model call). The DI example below is
the best showcase of the branching view.

## `explain` — understand code

| file | what it shows | command |
|---|---|---|
| [`explain-agent-getaction.md`](explain-agent-getaction.md) | **Deep explain, 3 levels / 6 methods** — the structured-output flow (`GetAction → ChatAsync → BuildOllama/BuildOpenAI`). Each method explained, then an end-to-end synthesis, an "In plain words" recap, and the **Call-flow** call-tree. | `explain --method "Agent.GetAction" --depth 3 --max-methods 9` |
| [`explain-deep-runasync.md`](explain-deep-runasync.md) | **Full-options, deep + wide: 12 methods / 3 levels** — the whole agent loop from `RunAsync` down. Every method explained, end-to-end synthesis, "In plain words", **and the Call-flow diagram**. The "it went through the whole code and summarised it" showcase. | `explain --method "Agent.RunAsync" --depth 3 --max-methods 12` |

## `trace` — find the call path

| file | what it shows | command |
|---|---|---|
| [`trace-with-bodies.md`](trace-with-bodies.md) | **One path, with the code between hops** — each method from its start to the call to the next hop, parameter names, `arg → param` mapping, the target's full body, then the **Call-flow** (vertical boxes). Deterministic (no model). | `trace -f RoslynIndex.cs -e Agent.cs --no-llm --with-bodies --repo-url …` |
| [`trace-with-bodies-annotated.md`](trace-with-bodies-annotated.md) | **Full-options trace** — code between hops, a "why" note per hop (`--annotate`), a final **Summary** + "In plain words" recap (`--summary`), and the **Call-flow** diagram. Everything a dev gets in one file. | `trace … --with-bodies --annotate --summary --repo-url …` |
| [`trace-agent-to-roslynindex.md`](trace-agent-to-roslynindex.md) | **All paths (brute force)** — every distinct way `Agent.cs` reaches `RoslynIndex.cs` (15 paths, each separated by a `---` rule), a closing summary, and the merged **Call-flow** tree. | `trace -f RoslynIndex.cs -e Agent.cs --all-paths --no-llm --summary --repo-url …` |
| [`trace-di-multiple-impls.md`](trace-di-multiple-impls.md) | **DI / interfaces, multiple implementations** — a call through an `INotifier` interface with 3 impls (+ a decorator); `--all-paths` lists all 5 distinct paths, and the **Call-flow** draws the fan-out as a branching tree (the best showcase). From [`../samples/di-playground`](../samples/di-playground). | `trace --from "NotificationService.Notify" --to "Audit.Record" --all-paths --no-llm` |
