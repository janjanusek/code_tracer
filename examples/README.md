# Examples

Real CodeTracer output, generated by running the tool **on its own source** (so every example is
reproducible from this repo). Each is committed exactly as produced (CPU, no GPU). A curated,
"everything on" set — one full **explain**, one full **trace**, one full **map**, the **DI** showcase,
plus the **same explain on a micro model** (`gemma3n:e2b`) so you can compare speed vs quality.

`explain` shows **each method's real source** under its heading (whole section indented by call-depth
via blockquotes, so the nesting is visible) followed by the explanation — code *and* prose, the way a
newcomer needs it.
Every result ends with an auto-generated **`## Call-flow`** diagram — a high-level map of what the
analysis found, drawn as **ASCII** (readable in any viewer) **and** **Mermaid** (renders as
graphics on GitHub / VS Code). It's deterministic (no extra model call).

| file | what it shows | command |
|---|---|---|
| [`explain-full-example.md`](explain-full-example.md) | **The full `explain` showcase** — deep + wide (12 methods, 3 levels) from `Agent.RunAsync` down. Each method's **real source** (whole section indented by call-depth via blockquotes, foldable `<details>`) + a step-by-step explanation, then an end-to-end synthesis, an **"In plain words"** recap, and the **Call-flow** diagram. Everything a newcomer needs, in one read. `gemma4:latest`, ~27 min. | `explain --method "Agent.RunAsync" --depth 3 --max-methods 12` |
| [`explain-gemma3n-e2b-example.md`](explain-gemma3n-e2b-example.md) | **Same explain, micro model** — identical command on **`gemma3n:e2b`** instead of `gemma4:latest`: **~3.4 min (≈7.8× faster)**, terser but still coherent. Read it next to the full example to judge a tiny local model on deep code understanding. | `explain --method "Agent.RunAsync" --depth 3 --max-methods 12 --model gemma3n:e2b` |
| [`trace-full-example.md`](trace-full-example.md) | **The full `trace` showcase** — the path with the **code between every hop**, a **"why" note per hop** (`--annotate`), a final **Summary** + **"In plain words"** recap (`--summary`), and the **Call-flow** diagram (boxes + Mermaid). | `trace -f RoslynIndex.cs -e Agent.cs --with-bodies --annotate --summary` |
| [`trace-di-multiple-impls.md`](trace-di-multiple-impls.md) | **DI / interfaces, multiple implementations** — a call through an `INotifier` interface with 3 impls (+ a decorator); `--all-paths` lists all 5 distinct paths, and the **Call-flow** draws the fan-out as a branching tree. From [`../samples/di-playground`](../samples/di-playground). | `trace --from "NotificationService.Notify" --to "Audit.Record" --all-paths --no-llm` |
| [`map-full-example.md`](map-full-example.md) | **The full `map` showcase** — both directions from `Agent.RunAsync`: **downstream** (the whole app call tree it reaches) and **upstream** (who reaches it — impact). Deterministic, **0 model calls**; each direction as an ASCII tree **and** a Mermaid graph. | `map --method "Agent.RunAsync"` |

Each example notes its real run-time and token usage. See [`../MANUAL.md`](../MANUAL.md) for the
full reference.
