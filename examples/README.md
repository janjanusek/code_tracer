# Examples

Real CodeTracer output, generated by running the tool **on its own source** (so every example
is reproducible from this repo). Each is committed exactly as produced (`gemma4:latest`, CPU).

## `explain` — understand code

| file | what it shows | command |
|---|---|---|
| [`explain-agent-getaction.md`](explain-agent-getaction.md) | **Deep explain, 3 levels / 6 methods** — the structured-output flow (`GetAction → ChatAsync → BuildOllama/BuildOpenAI`). Each method explained, then an end-to-end synthesis + an "In plain words" recap. | `explain --method "Agent.GetAction" --depth 3 --max-methods 9` |
| [`explain-deep-runasync.md`](explain-deep-runasync.md) | **Deeper explain, 12 methods / 3 levels** — the whole agent loop from `RunAsync` down. The "it went through the whole code and summarised it" showcase. | `explain --method "Agent.RunAsync" --depth 3 --max-methods 12` |

## `trace` — find the call path

| file | what it shows | command |
|---|---|---|
| [`trace-with-bodies.md`](trace-with-bodies.md) | **One path, with the code between hops** — each method from its start to the call to the next hop, parameter names, `arg → param` mapping, the target's full body. Deterministic (no model). | `trace -f RoslynIndex.cs -e Agent.cs --no-llm --with-bodies --repo-url …` |
| [`trace-with-bodies-annotated.md`](trace-with-bodies-annotated.md) | **The same, plus AI** — a short "why" note per hop (`--annotate`) and a final **Summary** + "In plain words" recap (`--summary`). | `trace … --annotate --summary --repo-url …` |
| [`trace-agent-to-roslynindex.md`](trace-agent-to-roslynindex.md) | **All paths (brute force)** — every distinct way `Agent.cs` reaches `RoslynIndex.cs` (15 paths, each separated by a `---` rule), with a closing summary. | `trace -f RoslynIndex.cs -e Agent.cs --all-paths --no-llm --summary --repo-url …` |
| [`trace-di-multiple-impls.md`](trace-di-multiple-impls.md) | **DI / interfaces, multiple implementations** — a call through an `INotifier` interface with 3 impls (+ a decorator); `--all-paths` lists all 5 distinct paths to the target. From [`../samples/di-playground`](../samples/di-playground). | `trace --from "NotificationService.Notify" --to "Audit.Record" --all-paths --no-llm` |

Each example notes its real run-time and token usage. See [`../MANUAL.md`](../MANUAL.md) for the
full reference.
